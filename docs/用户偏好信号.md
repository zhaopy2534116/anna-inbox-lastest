# 用户偏好信号

## 两级偏好信号模型

### 信号层级

```
强信号：dont_prioritize（用户明确表达）
  → 直接写入 SnoozePrefs.senders / SnoozePrefs.threads
  → Phase 2 prompt 中告知 LLM："严格标准，只保留必须用户亲自处理的事项"

弱信号：no_action_needed（用户行为推断）
  → 写入 LearningRecord（已有数据模型，当前仅用于 /ask 模式）
  → Phase 2 prompt 中告知 LLM："该 sender 历史上被用户标记为无需处理，
     仍可生成卡片，但标注为 lower_priority"
  → 不直接写入 SnoozePrefs，避免一次操作产生过度泛化
```

### 为什么 `no_action_needed` 不能直接写 SnoozePrefs

`dont_prioritize` 是用户**主动**点击 snooze 菜单第三项，意图明确——"以后这类事都降权"。

`no_action_needed` 是用户在抽屉里**处理完当前卡片**后的决策。用户可能的心理模型：

| 场景 | 用户真实意图 | 是否应泛化 |
|------|------------|-----------|
| 一封确实不重要的通知 | "这类事不用提醒我" | ✅ 可以 |
| 同事发的邮件，今天刚好不需要回复 | "这次不用，下次可能还要" | ❌ 不能 |
| 已经在别处处理了，但点错了按钮 | "处理过了"（本该点 handled manually） | ❌ 不能 |

如果直接把 `no_action_needed` 等同于 `dont_prioritize` 写入 SnoozePrefs，后两种场景会产生错误泛化——同事下次的重要邮件也会被降权。

### 具体设计

**`main.py` 中 `record_card_decision("no_action_needed")` 的处理：**

```python
if decision == "no_action_needed":
    # 1. 标记卡片为 resolved（现有逻辑）
    await update_card_status(mailbox, card_id, "resolved", "no_action_needed")

    # 2. 写入弱信号：LearningRecord
    card = await get_card_by_id(mailbox, card_id)
    if card:
        await append_learning(
            pattern=card.original.from_addr,
            action="no_action_needed",
        )
```

**`LearningRecord` 数据结构（已有，无需改动）：**

```python
@dataclass
class LearningRecord:
    pattern: str = ""        # sender email
    action: str = ""         # "no_action_needed"
    learnt_at: str = ""      # timestamp
```

**`judgment.py` prompt 注入时的区分：**

```python
def _render_snooze_prefs_context(prefs, learning_records):
    parts = []

    # 强信号：SnoozePrefs（用户明确 dont_prioritize）
    if prefs and (prefs.senders or prefs.threads):
        parts.append("## User Attention Preferences")
        parts.append("STRICT deprioritization: only surface if user MUST act.")
        # ...

    # 弱信号：LearningRecord 中的 no_action_needed 频率
    sender_counts = _count_no_action_needed_by_sender(learning_records)
    if sender_counts:
        frequent = {s: c for s, c in sender_counts.items() if c >= 2}
        if frequent:
            parts.append("## Historical Preferences")
            parts.append("The user has marked items from these senders as 'no action needed' multiple times:")
            for sender, count in frequent.items():
                parts.append(f"  - {sender} ({count} times)")
            parts.append("Consider showing these as lower_priority rather than main items, unless the email is clearly urgent.")

    return "\n".join(parts)
```

**关键设计决策：频次门槛**

`no_action_needed` 累计 2 次以上才在 prompt 中提及，单次不触发。这样：
- 一次性的"这次不用处理"不会产生泛化
- 同一 sender 多次被标记 → 形成可信的弱信号 → 降为 lower_priority（而非直接 ignore）
- 与 `dont_prioritize` 的"严格标准"形成两级梯度

### 信号梯度总结

| 信号 | 来源 | 强度 | Prompt 中的效果 | 写入位置 |
|------|------|------|----------------|---------|
| `dont_prioritize` | Snooze 菜单主动选择 | 强 | 严格标准：只保留必须用户处理的事项 | `SnoozePrefs` |
| `no_action_needed` ×2+ | 抽屉处理决策累积 | 弱 | 降为 lower_priority，紧急时仍可进入 main | `LearningRecord` |
| `no_action_needed` ×1 | 单次标记 | 无 | 不作提示 | `LearningRecord`（存档） |
