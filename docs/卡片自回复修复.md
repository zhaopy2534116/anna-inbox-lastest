# 修复"自己回复自己"卡片及持久化问题

## 问题概述

`kate@anna.partners` 邮箱当前 52 张活跃卡片中，29 张（56%）是"Kate Zhou 给你发了合作提案，请回复她"——但 Kate Zhou 就是用户本人。这些卡片来自用户自己发出的 SENT 邮件，被系统误当作待处理事项。

### 发现的问题清单

| # | 问题 | 数据 |
|---|------|------|
| 1 | SENT 邮件生成"回复自己"卡片 | 29/52 张卡片涉及 Kate Zhou，全部是用户自己发出的 |
| 2 | Gmail 查询未区分 INBOX/SENT/DRAFT | 缓存中 107/357 (30%) 来自用户本人，其中 70 条 SENT |
| 3 | `_dedupe_by_thread` 方向盲区 | 4 个混合线程中 SENT 胜出，丢弃了对方的 INBOX 消息 |
| 4 | LLM 不知道用户身份 | prompt 未告知 `kate@anna.partners` 就是用户本人 |
| 5 | DRAFT 被当作普通邮件 | 1 条 DRAFT 草稿会生成"请回复 Kate Zhou"卡片 |
| 6 | 低价值卡片冗余 | 21/52 (40%) 是 low/ignore 优先级 |
| 7 | 重复卡片 | 4x "Kate Zhou's Anna AI partnership proposal" 等 |

## 防线设计（三层）

```
Gmail API
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ L1  查询层：按策略区分                                │
│    default_secretary:   in:inbox                    │
│    creator_opportunity: -in:sent -in:draft           │
│    security_billing:    -in:sent -in:draft           │
│                                                      │
│    误杀分析：                                         │
│    - default_secretary 只看收件箱 → 符合"秘书"定位    │
│    - 另两种用 -in:sent 而非 in:inbox → 归档邮件不丢失 │
└─────────────────────────────────────────────────────┘

### L1 三种策略模式的查询范围详解

| 模式 | 过滤方式 | 扫描范围 |
|------|---------|---------|
| `default_secretary` | `in:inbox` | 仅收件箱 |
| `creator_opportunity` | `-in:sent -in:draft` | 全邮箱（收件箱 + 归档 + 各类 label + 社交/促销/更新 tab） |
| `security_billing` | `-in:sent -in:draft` | 全邮箱（同上，UPDATES 分类权重提升为 high） |

**`default_secretary` — `in:inbox`**

职责是"收件箱里有什么需要处理"。只看收件箱，不扫归档、不发散。

**`creator_opportunity` — `-in:sent -in:draft`**

职责是"全邮箱搜索合作线索"。90 天窗口内，不管邮件在收件箱、归档还是某个 label 下。
- 排除 `-in:sent`：用户自己发出的 outreach 不需要"提醒自己回复"
- 排除 `-in:draft`：没发出去的草稿不是合作线索
- 归档的合作邮件仍然有价值——90 天前的线索值得重新挖出来

**`security_billing` — `-in:sent -in:draft`**

职责是"全邮箱搜索安全/账单风险"。90 天窗口内，`label_weights` 中 `CATEGORY_UPDATES: high`——账单/安全通知常落在 UPDATES 分类。
- 排除自己发出的：用户不会给自己发安全告警
- 归档的旧告警也值得重新关注，所以不用 `in:inbox`

**与 Brief 的 L2 方向过滤的关系**

这两个模式没有 `_dedupe_by_thread` 那样的方向过滤。因为它们的定位不是"找待回复"，而是"找线索/风险"。即使用户已经回复过某个线程（SENT 最新），旧的 INBOX 消息作为线索仍有参考价值。L2 方向过滤仅对 `default_secretary` 生效。

    │
    ▼
┌─────────────────────────────────────────────────────┐
│ L2  去重层：方向感知的线程去重                         │
│    每个线程保留：最新 INBOX > 最新 DRAFT > 丢弃        │
│                                                      │
│    误杀分析：                                         │
│    - 丢弃的消息在 Phase 2 的 thread context 中可恢复   │
│    - 仅决定"锚点消息"，LLM 评估时看完整线程            │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ L3  LLM 层：告知用户身份                               │
│    Phase 1 + Phase 2 prompt 中增加：                  │
│    "Mailbox owner: kate@anna.partners.               │
│     Match by EMAIL ADDRESS, not display name."       │
│                                                      │
│    误杀分析：                                         │
│    - 按邮箱精确匹配 → "Kate (Meetup)" 不会被误判       │
│    - DRAFT 从自发的正确识别为"未发送草稿"              │
└─────────────────────────────────────────────────────┘
    │
    ▼
  Phase 2 → Card → Persist
```

L4（candidate filter）砍掉——L1+L2 之后 SENT 消息不可能再到达，完全冗余。

### 步骤 2：L2 — 线程去重方向感知

**文件**: `new/executas/tool-zhaopy-anna-inbox-tytvcy26/src/mail_agent/pipeline.py` (L69-L89)

当前实现：
```python
def _dedupe_by_thread(messages: list[Any]) -> list[Any]:
    """Keep only the latest message per thread."""
    thread_best: dict[str, Any] = {}
    for m in messages:
        tid = m.thread_id or m.message_id
        existing = thread_best.get(tid)
        if existing is None:
            thread_best[tid] = m
            continue
        try:
            if int(m.internal_date or 0) > int(existing.internal_date or 0):
                thread_best[tid] = m
        except (TypeError, ValueError):
            pass
    return list(thread_best.values())
```

改为：
```python
def _dedupe_by_thread(messages: list[Any]) -> list[Any]:
    """每个线程保留一条消息，优先级：最新 INBOX > 最新 DRAFT > 丢弃 SENT。

    纯 SENT 线程（用户发出但无回复）直接丢弃——用户已处理过。
    DRAFT 仅在没有 INBOX 时保留——表示用户开始写但未发送。
    """
    from collections import defaultdict

    thread_msgs: dict[str, list[Any]] = defaultdict(list)
    for m in messages:
        tid = m.thread_id or m.message_id
        thread_msgs[tid].append(m)

    def _max_by_date(msgs: list[Any]) -> Any:
        return max(msgs, key=lambda m: int(getattr(m, 'internal_date', '0') or 0))

    result: list[Any] = []
    for msgs in thread_msgs.values():
        inbox_msgs = [m for m in msgs if 'INBOX' in (getattr(m, 'label_ids', None) or [])]
        draft_msgs = [m for m in msgs if 'DRAFT' in (getattr(m, 'label_ids', None) or [])]

        if inbox_msgs:
            result.append(_max_by_date(inbox_msgs))
        elif draft_msgs:
            result.append(_max_by_date(draft_msgs))
        # else: 纯 SENT → 丢弃

    return result
```
