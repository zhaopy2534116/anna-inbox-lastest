# 分类筛选标签 — 实施方案

> 目标：首页顶部增加 `All / Needs reply / Needs review / Cleanup` 分类标签栏
> 策略：后端透传 `item_type`，前端映射为 3 个用户可见分类

---

## 1. 分类映射

```
后端 item_type                   →  前端 category     →  标签文案
─────────────────────────────────────────────────────────────────
reply_required                  →  "reply"           →  Needs reply
confirmation_required           →  "reply"           →  Needs reply
business_or_creator_thread      →  "reply"           →  Needs reply
security_risk                   →  "review"          →  Needs review
billing_or_subscription         →  "review"          →  Needs review
account_notice                  →  "review"          →  Needs review
low_value_cleanup               →  "cleanup"         →  Cleanup
unknown / 缺失                   →  "review"          →  Needs review（兜底）
```

## 2. 改动清单

### 2.1 后端 — `storage_types.py`

`PersistentCard` 新增一个字段：

```python
@dataclass
class PersistentCard:
    ...
    item_type: str = ""          # 新增：从 judgment 透传的分类信号
```

位置：在 `priority: str = "medium"` 之后加一行。

### 2.2 后端 — `card_service.py`

构建 `PersistentCard` 时传入 `item_type`。当前构建位置在 `_build_card()` 函数（约第 76 行）：

```python
# 修改前
return PersistentCard(
    card_id=...,

# 修改后
return PersistentCard(
    card_id=...,

```

`item_type` 变量已在函数中计算（第 212-215 行），只需在构造时传入。

### 2.3 后端 — `storage_ops.py`

序列化 `PersistentCard` 为 dict 时，确保 `item_type` 被包含。当前 `_card_to_dict()` 或等效序列化逻辑中加上：

```python
"item_type": card.item_type or "",
```

### 2.4 前端 — `app.js`

**a) 新增分类常量 + 映射函数**

```js
const CATEGORY_TABS = [
  { id: "all", label: "All" },
  { id: "reply", label: "Needs reply" },
  { id: "review", label: "Needs review" },
  { id: "cleanup", label: "Cleanup" },
];

const ITEM_TYPE_CATEGORY = {
  reply_required: "reply",
  confirmation_required: "reply",
  business_or_creator_thread: "reply",
  security_risk: "review",
  billing_or_subscription: "review",
  account_notice: "review",
  low_value_cleanup: "cleanup",
};

function cardCategory(card) {
  return ITEM_TYPE_CATEGORY[card.item_type] || "review";
}
```

**b) `state` 新增字段**

```js
resultFilter: "all",  // 当前选中的分类标签
```

**c) 修改 `renderStart()` — 在 assistant-card 下方插入标签栏**

```html
<div class="result-filter-row">
  <div class="category-segment" role="tablist">
    ${CATEGORY_TABS.map(tab => `
      <button class="category-tab ${activeFilter === tab.id ? "is-active" : ""}"
              data-result-filter="${tab.id}">
        ${tab.label} ${filteredCount(tab.id)}
      </button>
    `).join("")}
  </div>
</div>
```

**d) 新增 `filteredCards(category)` — 按分类过滤**

```js
function filteredCards(category) {
  const visible = visibleCards();
  if (category === "all") return visible;
  return visible.filter(card => cardCategory(card) === category);
}

function filteredCount(category) {
  return filteredCards(category).length;
}
```

**e) 修改卡片渲染区域 — 按 `resultFilter` 渲染**

```js
const displayCards = filteredCards(state.resultFilter);
// 替换原来的 cards.map(renderAttentionCard)
```

**f) 点击事件 — `data-result-filter`**

```js
const filterTab = event.target.closest("[data-result-filter]");
if (filterTab) {
  state.resultFilter = filterTab.dataset.resultFilter;
  render();
  return;
}
```

## 3. 用户交互

```
首页（有卡片时）
┌─────────────────────────────────────────┐
│  I found 3 things worth your attention. │
│                                         │
│  ┌──────────────────────────────────┐   │
│  │ All 3  Needs reply 2  Reviews 1 │   │  ← 分类标签栏
│  └──────────────────────────────────┘   │
│                                         │
│  ┌─ Christopher is waiting... ────────┐ │
│  │  ...                               │ │  ← 仅显示当前分类的卡片
│  └────────────────────────────────────┘ │
│  ┌─ Jane's assignment... ─────────────┐ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 默认选中 "All"
- 点击标签切换分类，仅渲染对应卡片
- 每个标签右侧显示该分类的卡片数量
- Cleanup 分类的卡片渲染时可考虑折叠样式（用现有的 `lowerPriority` 区域）

## 4. 范围外

- Cleanup 分类不单独展示为 lower-priority 折叠区，而是和其他分类一样平铺在标签栏中。现有的 `lowerPriorityOpen` 折叠逻辑保留，但仅在没有分类筛选时使用。
- 标签栏仅在 `visibleCards().length > 0` 时显示
- 首扫页面（`isFresh`）不显示标签栏
