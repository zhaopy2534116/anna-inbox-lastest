# 扫描进度显示优化

> 2026-06-03

## 问题

扫描过程中每个阶段复用同一个 `scanReviewed` 数字，在 "Connecting mailbox" 阶段始终显示 `0 emails reviewed`，后续阶段数字也不随阶段变化，用户感觉扫描卡住了。

## 方案

每个阶段使用该阶段特有的进度文本，不用一个数字通吃。

## 效果对比

| 阶段 | 改前 | 改后 |
|------|------|------|
| Connecting mailbox | `0 emails reviewed` | `Connecting to Gmail…` |
| 邮件加载完成 | `20 emails reviewed` | `20 emails loaded` |
| 过滤去重 | `20 emails reviewed` | `5 skipped, 15 new` |
| Phase1 分类中 | `20 emails reviewed` | `Classifying 15 emails…` |
| Phase1 完成 | `20 emails reviewed` | `8 candidates, 7 low-priority` |
| 读上下文 | `20 emails reviewed` | `Reading context 3/8` |
| 评估中 | `20 emails reviewed` | `Evaluating 2/8` |
| 生成计划 | `20 emails reviewed` | `Building action plan…` |

## 实现

`app.js` 新增 `scanProgressLabel(stage, progress)` 函数，按阶段返回对应文本。`state.scanReviewed` 替换为 `state.scanStage` + `state.scanProgress`，`renderScanning` 和 `renderBottomBar` 使用新标签。
